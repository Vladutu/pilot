# Pilot Release Automation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** One command on the Mac — `scripts/release.sh 0.2.0` — runs tests, builds a release-signed APK, and publishes it as the *latest* GitHub release on the private `Vladutu/pilot` repo so Obtainium auto-updates both devices.

**Architecture:** Restore the (incomplete) Gradle wrapper so CLI builds work; make `versionName`/`versionCode` arg-driven via Gradle `-P` properties; switch the `release` build type from debug-signing to a dedicated keystore sourced from `~/.gradle/gradle.properties`; add a `release.sh` that orchestrates test → assemble → tag → `gh release create --latest`.

**Tech Stack:** Bash, Gradle 8.9 (Kotlin DSL), Android Gradle Plugin, `gh` CLI, `keytool`, Obtainium (devices).

## Task 1: Restore the Gradle wrapper

The repo has `gradle/wrapper/gradle-wrapper.properties` (Gradle 8.9) but no `gradlew`/`gradlew.bat`/`gradle-wrapper.jar`. CLI builds need them.

> **Lesson learned (2026-06-07):** the first version of this script downloaded Gradle to a
> temp dir and ran `gradle wrapper` from there — which baked the soon-deleted temp path into
> `gradle-wrapper.properties` and broke every `./gradlew` call (`NoSuchFileException` on a
> `/var/folders/.../T/...` path). Fix: regenerate from a *permanently installed* Gradle and
> pin the distribution URL explicitly. Stale Gradle daemons bound to the dead dist also need
> `gradle --stop`.

**Files:**
- Create: `scripts/bootstrap-wrapper.sh`
- Generated on Mac: `gradlew`, `gradlew.bat`, `gradle/wrapper/gradle-wrapper.jar`, `gradle/wrapper/gradle-wrapper.properties`

- [ ] **Step 1 (EDIT): Write `scripts/bootstrap-wrapper.sh`**

```bash
#!/usr/bin/env bash
set -euo pipefail

# Regenerate the Gradle wrapper (gradlew, gradlew.bat, gradle-wrapper.jar,
# gradle-wrapper.properties) pinned to GRADLE_VERSION. Run once, then commit the
# generated files. Requires a normally-installed Gradle on PATH:
#   sdk install gradle 8.9      # SDKMAN
#   brew install gradle         # Homebrew

GRADLE_VERSION="8.9"
DIST_URL="https://services.gradle.org/distributions/gradle-${GRADLE_VERSION}-bin.zip"
PROPS="gradle/wrapper/gradle-wrapper.properties"

cd "$(dirname "$0")/.."

# Find a gradle: PATH first, then SDKMAN's (its init isn't sourced in scripts).
if command -v gradle >/dev/null 2>&1; then
  GRADLE_BIN="gradle"
elif [ -x "$HOME/.sdkman/candidates/gradle/current/bin/gradle" ]; then
  GRADLE_BIN="$HOME/.sdkman/candidates/gradle/current/bin/gradle"
else
  echo "error: no 'gradle' found. Install it first, e.g.:" >&2
  echo "  sdk install gradle ${GRADLE_VERSION}   # SDKMAN" >&2
  echo "  brew install gradle                    # Homebrew" >&2
  exit 1
fi

"$GRADLE_BIN" --stop >/dev/null 2>&1 || true   # drop any stale daemon before regenerating
"$GRADLE_BIN" wrapper --gradle-distribution-url "$DIST_URL"

if ! grep -q "^distributionUrl=https\\\\://services.gradle.org/distributions/gradle-${GRADLE_VERSION}-bin.zip" "$PROPS"; then
  echo "error: unexpected distributionUrl in $PROPS:" >&2
  grep "^distributionUrl=" "$PROPS" >&2
  exit 1
fi

echo "Done. Commit: gradlew gradlew.bat gradle/wrapper/gradle-wrapper.jar $PROPS"
```

Run: `chmod +x scripts/bootstrap-wrapper.sh`

- [ ] **Step 3 (MAC): Ensure a Gradle is installed, then generate the wrapper**

Run: `sdk install gradle 8.9` (or `brew install gradle`), then `./scripts/bootstrap-wrapper.sh`
Expected: ends with `Done. Commit: ...`, and `gradlew` / `gradlew.bat` / `gradle-wrapper.jar` now exist.

## Task 3: Release keystore (committed in-repo)

> **Decision changed 2026-06-07:** the owner chose to commit the keystore + password into
> the private repo for zero-setup builds. The keystore is generated once and committed;
> `signing.properties` holds the (non-secret) password. Build falls back to debug signing
> only if those files are absent.

**Files:**
- Create (committed): `keystore/pilot-release.jks`, `keystore/signing.properties`
- Modify: `app/build.gradle.kts`

- [ ] **Step 1 (EDIT): Generate the keystore (once)**

```bash
mkdir -p keystore
keytool -genkeypair -v \
  -keystore keystore/pilot-release.jks \
  -storetype PKCS12 -alias pilot \
  -keyalg RSA -keysize 4096 -validity 10000 \
  -dname "CN=Pilot, O=vladutu, C=RO" \
  -storepass <local-password> -keypass <local-password>
```

- [ ] **Step 2 (EDIT): Write `keystore/signing.properties`**

```properties
storeFile=pilot-release.jks
storePassword=<local-password>
keyAlias=pilot
keyPassword=<local-password>
```

- [ ] **Step 3 (EDIT): Add the `release` signingConfig that reads the in-repo file**

In `app/build.gradle.kts`, add `import java.util.Properties` at the top, then add a
`signingConfigs` block immediately before `buildTypes` inside `android { }`:

```kotlin
    val releaseSigningProps = rootProject.file("keystore/signing.properties")
    signingConfigs {
        create("release") {
            if (releaseSigningProps.exists()) {
                val props = Properties().apply {
                    releaseSigningProps.inputStream().use { load(it) }
                }
                storeFile = rootProject.file("keystore/${props.getProperty("storeFile")}")
                storePassword = props.getProperty("storePassword")
                keyAlias = props.getProperty("keyAlias")
                keyPassword = props.getProperty("keyPassword")
            }
        }
    }
```

Then in `buildTypes { release { ... } }` replace the debug `signingConfig` line with:

```kotlin
            signingConfig = if (releaseSigningProps.exists()) {
                signingConfigs.getByName("release")
            } else {
                signingConfigs.getByName("debug") // fallback if keystore/signing.properties is absent
            }
```

- [ ] **Step 4 (MAC): Build a release APK and confirm the new signer**

```bash

"$(ls -d ~/Library/Android/sdk/build-tools/*/ | sort -V | tail -1)apksigner" verify --print-certs \
  app/build/outputs/apk/release/app-release.apk
```
Expected: `BUILD SUCCESSFUL`, and a `V2 Signer: certificate DN: CN=Pilot, O=vladutu, C=RO` line

Scheme v2/v3 with no legacy v1 JAR signature, so `keytool -jarfile` reports "Not a signed jar file"
even on a correctly signed APK.

## Task 4: The release script

**Files:**
- Create: `scripts/release.sh`

- [ ] **Step 1 (EDIT): Write `scripts/release.sh`**

```bash
#!/usr/bin/env bash
set -euo pipefail

# Pilot release: test -> build release-signed APK -> publish as the latest
# GitHub release. Runs on the Mac. Requires: JDK 17/21, Android SDK, gh (authed),
# the release keystore props in ~/.gradle/gradle.properties, and a committed
# Gradle wrapper (see scripts/bootstrap-wrapper.sh).
#
# Usage: scripts/release.sh <MAJOR.MINOR.PATCH>

REPO="Vladutu/pilot"
die() { echo "error: $*" >&2; exit 1; }

# --- args / version math (runs before any side effects) ---
VERSION="${1:-}"
[ -n "$VERSION" ] || die "usage: scripts/release.sh <MAJOR.MINOR.PATCH>"
if [[ ! "$VERSION" =~ ^([0-9]+)\.([0-9]+)\.([0-9]+)$ ]]; then
  die "version must be semver MAJOR.MINOR.PATCH (got '$VERSION')"
fi
MAJOR="${BASH_REMATCH[1]}"; MINOR="${BASH_REMATCH[2]}"; PATCH="${BASH_REMATCH[3]}"
if [ "$MINOR" -ge 100 ] || [ "$PATCH" -ge 100 ]; then
  die "minor and patch must each be < 100 (versionCode scheme)"
fi
VCODE=$(( MAJOR * 10000 + MINOR * 100 + PATCH ))
TAG="v$VERSION"

cd "$(dirname "$0")/.."

# --- preflight ---
[ -z "$(git status --porcelain)" ] || die "working tree is dirty; commit or stash first"
if git rev-parse "$TAG" >/dev/null 2>&1; then die "tag $TAG already exists locally"; fi
if git ls-remote --exit-code --tags origin "$TAG" >/dev/null 2>&1; then
  die "tag $TAG already exists on origin"
fi

command -v gh >/dev/null 2>&1 || die "gh not found; install: brew install gh && gh auth login"
gh auth status >/dev/null 2>&1 || die "gh not authenticated; run: gh auth login"

if [ -z "${JAVA_HOME:-}" ] || ! "${JAVA_HOME:-/nonexistent}/bin/java" -version >/dev/null 2>&1; then
  JBR="/Applications/Android Studio.app/Contents/jbr/Contents/Home"
  [ -d "$JBR" ] || die "set JAVA_HOME to a JDK 17/21 (Android Studio JBR not found at $JBR)"
  export JAVA_HOME="$JBR"
fi
echo "Using JAVA_HOME=$JAVA_HOME"

if ! grep -qs "RELEASE_STORE_FILE" "$HOME/.gradle/gradle.properties" \
   && [ -z "${ORG_GRADLE_PROJECT_RELEASE_STORE_FILE:-}" ]; then
  die "release signing props missing; add RELEASE_STORE_FILE/PASSWORD/KEY_ALIAS/KEY_PASSWORD to ~/.gradle/gradle.properties"
fi

if [ ! -x "./gradlew" ] || [ ! -f "gradle/wrapper/gradle-wrapper.jar" ]; then
  die "Gradle wrapper missing; run scripts/bootstrap-wrapper.sh first"
fi

# --- test ---
echo "==> unit tests"
./gradlew --no-daemon testReleaseUnitTest

# --- build ---
echo "==> assembleRelease ($VERSION / code $VCODE)"
./gradlew --no-daemon assembleRelease -PversionName="$VERSION" -PversionCode="$VCODE"

SRC_APK="app/build/outputs/apk/release/app-release.apk"
[ -f "$SRC_APK" ] || die "APK not found at $SRC_APK"
OUT_APK="pilot-$VERSION.apk"
cp "$SRC_APK" "$OUT_APK"

# --- push, tag + publish ---
echo "==> push branch + tag + GitHub release"
git push origin HEAD
git tag -a "$TAG" -m "Pilot $VERSION"
git push origin "$TAG"
gh release create "$TAG" \
  --repo "$REPO" \
  --latest \
  --title "Pilot $VERSION" \
  --generate-notes \
  "$OUT_APK"

rm -f "$OUT_APK"
echo "==> released $TAG -> https://github.com/$REPO/releases/latest"
```

- [ ] **Step 2 (EDIT): Make it executable**

Run: `chmod +x scripts/release.sh`

````markdown
# Releasing Pilot

## One-time setup (Mac)

1. **Gradle wrapper** (if `gradlew` is missing): `./scripts/bootstrap-wrapper.sh`
2. **Release keystore:**
   ```bash
   mkdir -p ~/keystores
   keytool -genkeypair -v -keystore ~/keystores/pilot-release.jks \
     -storetype PKCS12 -alias pilot -keyalg RSA -keysize 4096 -validity 10000 \
     -dname "CN=Pilot, O=vladutu, C=RO"
   ```
   Use the same store/key password.
3. **Signing creds** in `~/.gradle/gradle.properties`:
   ```properties
   RELEASE_STORE_FILE=/Users/geo/keystores/pilot-release.jks
   RELEASE_STORE_PASSWORD=<password>
   RELEASE_KEY_ALIAS=pilot
   RELEASE_KEY_PASSWORD=<password>
   ```
4. **gh CLI:** `brew install gh && gh auth login`

> The keystore lives only on this Mac and is never committed. Back it up — losing it
> means future updates can't install over installed copies (forces uninstall/reinstall).

## Cut a release

```bash
./scripts/release.sh 0.2.0
```
Runs tests, builds a release-signed APK, tags `v0.2.0`, and publishes it as the latest
release on `Vladutu/pilot` with `pilot-0.2.0.apk` attached.

`versionCode` = `major*10000 + minor*100 + patch` (so `0.2.0` -> `200`). Keep minor/patch < 100.

## Obtainium (each device: phone + car box)

1. Install Obtainium (from its GitHub releases page).
2. Settings -> add a **fine-grained GitHub PAT**, repo access limited to `Vladutu/pilot`,
   permission **Contents: Read-only**.
3. Add App -> URL `https://github.com/Vladutu/pilot`:
   - APK filter regex: `pilot-.*\.apk`
   - Track only the latest release.
4. Obtainium then prompts to update whenever `release.sh` publishes a newer version.

> The signing switch to the dedicated keystore means the first new build won't install
> over an old debug-signed Pilot — uninstall the old app once on each device, then install.
````

## Self-review notes

- **Spec coverage:** wrapper restore (T1), arg-driven versioning (T2), dedicated keystore + debug fallback + gitignore (T3), `release.sh` test→build→tag→`--latest` (T4), Obtainium docs (T5), end-to-end incl. one-time reinstall + update proof (T6). All spec sections mapped.
- **Deviation from spec:** wrapper bootstrap is a dedicated one-time `bootstrap-wrapper.sh` verified against Gradle's *published* `.sha256`, instead of an inline per-run bootstrap with a hardcoded checksum. Same intent (verified bootstrap), more trustworthy.
- **Consistency:** `RELEASE_STORE_FILE`/`RELEASE_STORE_PASSWORD`/`RELEASE_KEY_ALIAS`/`RELEASE_KEY_PASSWORD`, alias `pilot`, APK name `pilot-<version>.apk`, and the `major*10000+minor*100+patch` formula are used identically across the build file, `release.sh`, and docs.
- **`set -e` safety:** tag-existence checks use `if ...; then die; fi` (not `&& die`) so the "tag absent" happy path doesn't trip `set -e`.
